# TUI Agent Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace the GUI-first direction with an OpenCode-style terminal agent that starts inside a course folder, remembers prior sessions, indexes sources and generated outputs, and writes reusable study artifacts back to that folder.

**Architecture:** Add a folder-workspace layer beside the existing app backend. The current directory becomes the course workspace, `.cram/` stores parsed state, memory, session logs, and indexes, while `cram-output/` stores user-facing Markdown artifacts. A Textual TUI provides a full-screen chat flow with slash commands; generated outputs are re-indexed as lower-priority references.

**Tech Stack:** Python 3.13, Textual for TUI, existing MinerU ingestion helpers, existing OpenAI-compatible LLM client, JSONL/Markdown storage for first version.
