# LLM First Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Wire free-text TUI questions to the existing OpenAI-compatible LLM client.

**Architecture:** Keep slash commands deterministic. For free text, `CommandRouter` creates or receives an LLM client, builds a study-agent prompt, calls the model, and converts configuration/request failures into user-facing messages.

**Tech Stack:** Python 3.13, `httpx`, existing `OpenAICompatibleClient`, `unittest`.

---

### Task 1: Router LLM Tests

**Files:**
- Modify: `tests/test_tui_commands.py`

**Steps:**
1. Add a fake LLM client that records messages and returns a deterministic answer.
2. Add a test proving free text calls the LLM and returns its answer.
3. Add a test proving missing LLM configuration returns setup guidance instead of raising.
4. Run `python -m unittest tests.test_tui_commands -v` and verify the new tests fail.

### Task 2: Router LLM Implementation

**Files:**
- Modify: `app/backend/cram_app/commands.py`

**Steps:**
1. Add optional `llm` injection to `CommandRouter`.
2. Add default settings from `CRAM_LLM_BASE_URL`, `CRAM_LLM_MODEL`, and `CRAM_LLM_API_KEY`.
3. Replace the free-text placeholder with an LLM call.
4. Catch `LLMConfigurationError` and `LLMRequestError`.
5. Run `python -m unittest tests.test_tui_commands -v`.
